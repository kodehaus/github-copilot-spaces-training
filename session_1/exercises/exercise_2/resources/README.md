# Exercise 2 Resources: Workflow Scenarios

This folder contains realistic scenarios for the three workflows in Exercise 2. Use these scenarios to practice analyzing context gaps and engineering effective solutions.

## How to Use These Scenarios

1. **Read each scenario carefully** - Understand the situation and what's being asked
2. **Analyze the context gap** - What information is missing that would help the AI?
3. **Reference Exercise 1 resources** - Use the good_examples from Exercise 1 as templates
4. **Engineer your solution** - Add appropriate context to your Copilot Space
5. **Test and iterate** - Verify the AI provides better assistance

## Workflow-Specific Guidance

### Scenario 1: Debugging
**Key question:** What information helps identify the root cause?
- Error messages and stack traces
- State of the application when error occurred
- Related code that might be causing the issue

### Scenario 2: Feature Development
**Key question:** What ensures the new feature fits the existing system?
- Architectural patterns used in the project
- Examples of similar existing features
- Security and quality requirements

### Scenario 3: Code Review
**Key question:** What defines "good code" in this project?
- Style guides and conventions
- Project-specific patterns
- Common issues to watch for
