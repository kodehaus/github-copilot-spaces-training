# Facilitator Guide: Context Engineering GitHub Copilot Spaces

## Course Overview

**Duration:** 3 hours (180 minutes)

**Target Audience:** Developers using AI coding assistants

**Prerequisites:** Basic understanding of Git and GitHub, Experience with an IDE or code editor, Familiarity with AI-assisted coding tools (helpful but not required)

**Learning Framework:** Synergistic Instructional Design (TBR + TLC + VF)

## Learning Objectives

By the end of this course, participants will be able to:

1. **[Remember]** Identify and describe the core features and components of GitHub Copilot Spaces
   - Success criteria: Can name at least 5 core features without reference materials; Can explain the purpose of each feature in their own words; Can identify which feature applies to common scenarios
2. **[Understand]** Explain how context management works and why it matters for AI-assisted development
   - Success criteria: Can explain the relationship between context quality and AI output; Can describe what makes good vs. poor context; Can predict outcomes of different context strategies
3. **[Apply]** Set up and configure context-aware workspaces for real development scenarios
   - Success criteria: Can create a workspace with appropriate context sources; Can add and organize files, repositories, and instructions; Can validate that context is working as expected; Workspace passes quality checklist (coverage, relevance, organization)
4. **[Apply]** Use context engineering to accelerate common development workflows
   - Success criteria: Can engineer context for at least 3 different workflow types; Demonstrates measurable improvement in task completion time; Can adapt context strategy when initial approach is ineffective; Provides evidence of improved AI assistance quality
5. **[Analyze]** Diagnose context-related issues and identify improvement opportunities
   - Success criteria: Can audit a workspace and identify 3+ areas for improvement; Can explain the impact of identified issues on AI effectiveness; Can prioritize improvements based on expected impact; Recommendations align with documented best practices
6. **[Evaluate]** Assess the quality and effectiveness of context engineering approaches
   - Success criteria: Can compare multiple approaches using clear criteria; Can justify context decisions with evidence and reasoning; Can identify tradeoffs and explain when each approach is appropriate; Recommendations demonstrate understanding of security, performance, and maintainability
7. **[Create]** Design and implement a comprehensive context engineering strategy for a real-world project
   - Success criteria: Strategy addresses all aspects: setup, organization, maintenance, and team collaboration; Implementation demonstrates integration of at least 5 key concepts; Solution is well-documented with clear rationale for design decisions; Evidence shows improved development outcomes (quality, speed, or both); Approach is sustainable and scalable for team adoption

## Course Structure

This course follows the **4Cs Framework**:
- **Connection (10%):** Activate prior knowledge and set context
- **Concepts (30%):** Introduce new knowledge and understanding
- **Concrete Practice (50%):** Hands-on application and skill building
- **Conclusions (10%):** Reflection, consolidation, and next steps

## Pedagogical Strategies

### Core Principles

This course is built on evidence-based pedagogical practices:

1. **Active Learning:** Participants spend 50% of time in hands-on practice
2. **Metabolic Pacing:** Sessions under 150 minutes with activities under 15 minutes
3. **Spaced Practice:** Multiple opportunities to practice each skill
4. **Authentic Context:** Real-world scenarios and problems
5. **Feedback Loops:** Immediate feedback through assessments

## Materials Checklist

### Required Materials

- GitHub account for each participant
- GitHub Copilot access (Business or Enterprise)
- VS Code or supported IDE
- Sample project repositories
- Whiteboard or shared digital workspace

## Tips for Successful Delivery

1. **Pre-Session Setup**
   - Test all live demos in advance
   - Verify participant access to tools
   - Prepare backup plans for technical issues

2. **During Sessions**
   - Start with connection activities to build rapport
   - Monitor energy levels and adjust pacing
   - Encourage questions and peer discussion
   - Take breaks every 60-90 minutes

3. **Common Challenges**
   - Participants with varied experience levels: Use pair programming
   - Technical setup issues: Have troubleshooting guide ready
   - Running over time: Prioritize hands-on practice over lecture
